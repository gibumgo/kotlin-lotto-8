## 기능 구현 목록

### 로또 구입 금액 입력

- [ ] 구입할 금액을 입력받는다.
- [ ] 입력 값이 숫자인지 검증한다.
- [x] 입력 값이 1,000원 단위인지 확인한다.
- [x] 입력 값이 음수인지 확인한다.
- [ ] 유효하지 않으면 `[ERROR]` 메시지를 출력하고 재입력한다.

### 로또 발행

- [x] 구매 가능한 로또 수만큼 로또 번호를 생성한다.
    - 구매 가능한 로또 수를 계산한다.
    - 로또 번호를 생성한다.
- [x] 1~45 범위에서 중복되지 않는 6개의 숫자를 선택한다.
- [x] 로또를 생성한다.
- [ ] 여러 로또를 모아 확인할 수 있도록 정리한다.
- [ ] 구매한 로또를 출력한다.
- [ ] 유효하지 않으면 `[ERROR]` 메시지를 출력하고 재입력한다.

### 당첨 번호 입력

- [ ] 6개의 당첨 번호를 입력받는다.
- [ ] 입력 값이 숫자인지 검증한다.
- [x] 범위 1~45를 벗어나지 않았는지 확인한다.
- [ ] 입력 번호 중 중복이 없는지 확인한다.
- [ ] 보너스 번호를 입력받는다.
- [x] 보너스 번호가 당첨 번호와 중복되지 않도록 확인한다.

### 당첨 결과 확인

- [ ] 모든 로또를 당첨 번호와 비교한다.
- [ ] 각 로또의 일치 개수를 확인한다.
- [x] 등수를 계산한다.
    - 유효하지 않는 일치 개수는 예외를 발생시킨다.
- [x] 등수를 결과에 기록한다.
- [ ] 모든 로또 결과를 집계한다.

### 수익률 계산 및 출력

- [ ] 총 상금을 계산한다.
- [ ] 수익률을 계산한다.
- [ ] 수익률을 소수점 둘째 자리에서 반올림한다.
- [ ] 출력 시 소수점 첫째 자리까지 표현한다. (ex. 총 수익률은 62.5%입니다.)
- [ ] 당첨 내역을 출력한다.
- [ ] 수익률을 출력한다.
- [ ] 로또 번호를 오름차순으로 정렬하여 출력한다.

### 예외 처리

- [ ] 입력값이 잘못되면 예외를 발생시킨다.
- [ ] 입력값이 잘못되면 재입력할 수 있다.
- [ ] `[ERROR]` 메시지를 출력하고 재입력한다.

## 프로그래밍 요구사항
- indent(인덴트, 들여쓰기) depth를 3이 넘지 않도록 구현한다. 2까지만 허용한다.
- 3항 연산자를 쓰지 않는다.
- 함수(또는 메서드)의 길이가 15라인을 넘어가지 않도록 구현한다.
- 함수(또는 메서드)가 한 가지 일만 하도록 최대한 작게 만들어라.
- 구현한 기능에 대한 단위 테스트를 작성한다. 단, UI(System.out, System.in, Scanner) 로직은 제외한다.
- `camp.nextstep.edu.missionutils`에서 제공하는`Randoms`및`Console`API를 사용하여 구현해야 한다.

## 구현 단계에서 추가 될 요구 사항
- 커밋 메시지를 의미 있게 작성
    - 제목은 명령조로
    - 본문은 어떻게보다 무엇을, 왜에 맞춰 작성하기
- 네이밍에 시간 투자
    - 이름을 통해 의도를 드러낸다
    - 축약하지 않는다
    - 문맥을 중복하는 이름을 자제한다
      `ex) 클래스 이름 = Order`
      `메서드 이름 = shipOrder X , order.ship()` 간결한 호출
- 코드 포매팅을 사용
- 기능 목록을 재검토한다
    - 구현해야 할 기능 목록을 중심으로 작성
    - 지속적 업데이트 (ex. 예외 상황)
- 변수 이름에 자료형은 사용하지 않는다
- 한 메서드가 한 가지 기능만 담당하게 한다
